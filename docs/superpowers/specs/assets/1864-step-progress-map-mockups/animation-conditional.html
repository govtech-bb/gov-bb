The rules behind the motion

  • Width + fade, ~450ms, eased. The node (and its connector) collapse to zero width so the line closes the gap smoothly; a soft pulse marks a newly-inserted step.
  • Respects prefers-reduced-motion. With it on, the node just cross-fades near-instantly, no slide, no pulse (try your OS setting).
  • Never yanks the current node. Here Income inserts after "Dependents" (the current step), so your position doesn't move. If a step inserts before the current one, we keep the current node anchored in view rather than letting the map jump under you.
  • Mobile equivalent: the vertical list does the same thing as a height expand/collapse instead of width.
