How it holds together

  1. 01main is the trunk — the single source of truth where every feature merges.
  2. 02Every merge is gated by CI — no direct pushes to main. Stability is proven by QA on the environments; PR review comes as the team matures.
  3. 03Deploys run in sequence — sandbox deploys on merge; staging triggers only once the sandbox deploy is fully green. No parallel environment deploys.
  4. 04Production is manual & windowed — deployed by hand, only after staging is proven.
  5. 05Environments are deploy targets. Never developed or hot-fixed on directly.

feature → main (CI-gated) → sandbox → (when green) staging → prod (windowed)
